## [13.04.2026] - Неделя 2: Настройка Python-окружения и проверка связи
**Статус:** Успешно. Базы данных доступны из кода.

**Что сделано:**
1. Настроено виртуальное окружение (`venv`) внутри WSL 2.
2. Создан `requirements.txt` с жесткой фиксацией версий (`psycopg2-binary==2.9.11`, `qdrant-client==1.17.1`, `python-dotenv==1.2.2`).

**Архитектурные решения (ADR):**
* *Фиксация версий:* Принято решение жестко фиксировать версии библиотек в `requirements.txt` (через `==`), чтобы избежать поломок при мажорных обновлениях зависимостей в будущем.
* *Структура папок:* Скрипт проверки вынесен в папку `scripts/`, чтобы не засорять папку `src/` (где будет лежать только чистая бизнес-логика и ядро).

**Проблемы и решения:**
* *Проблема:* Cursor не видел Python-окружение, не работали подсказки кода.
* *Решение:* Установлено расширение Python от Microsoft *специально для WSL* через интерфейс Cursor.

* *Проблема:* Cursor (в режиме WSL) блокировал скачивание расширений Python/Pylance из-за сетевых ограничений/багов маркетплейса.
* *Решение (Hardcore-way):* 
  1. Вручную скачаны `.vsix` пакеты (ms-python, pylance, debugpy) из open-vsx.org.
  2. Пакеты распакованы напрямую в `~/.cursor-server/extensions/` внутри WSL.
  3. Вручную пропатчены манифесты версий и обновлен `extensions.json` для обхода конфликтов совместимости Cursor.
* *Обновление среды:* Установлен стабильный Python 3.14.4. Старый `venv` полностью удален и пересоздан с нуля под новую версию ядра. Зависимости переустановлены через `pip install -r requirements.txt`.
* *Рефакторинг структуры:* Изначально тестовый скрипт БД был ошибочно помещен в ad-hoc папку `scripts/`. Проведен рефакторинг: папка удалена, скрипт переименован в `test_db_connection.py` и перенесен в каноничную директорию `tests/`. Правило: строгое соблюдение утвержденной файловой структуры, никаких дублирующих сущностей.
